Security and governance

Private by design - secure by default

Data boundary

  • No model training
  • Customer isolation (separate instance - no sharing)
  • Configurable retention (medd only stores what is required by the multimodal interface layer for compliance and function)

Access & oversight

  • Role-based access (learner/assessor/admin)
  • Admin governance (scenario approval, assessor assignment)
  • Optional transcript controls (on/off, redaction)

Audit & compliance

  • Audit logs (who did what, when)
  • Rubric versioning (which rules + content were used)
  • Exportable evidence accessible by admin role/end-user
